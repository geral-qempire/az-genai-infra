## 1. Alert
Percentage of search queries that were throttled for the search service.

- **Metric**: `ThrottledSearchQueriesPercentage`
- **Namespace**: `Microsoft.Search/searchServices`
- **Aggregation**: Average
- **Operator**: GreaterThan (fires when percentage exceeds threshold)
- **Evaluation Frequency**: 1 minute (PT1M)
- **Time Window**: 5 minutes (PT5M)
- **Severity**: 3

## 2. Usage
```hcl main.tf
module "throttled_search_pct_alert" {
  source = "./modules/ai-search-service/alerts/ThrottledSearchQueriesPercentage"

  # name defaults to alrt-thrpct-<search-service-name>
  resource_group_name = azurerm_resource_group.rg.name
  scopes              = [module.ai_search.search_service_id]

  # Default threshold is 10%
  # threshold = 10

  description = "Average throttled search queries percentage above 10% over 5 minutes"
  enabled     = true
  action_group_ids = []
}
```

## 3. Inputs
| Name | Type | Default | Required | Description |
|------|------|---------|:--------:|-------------|
| `name` | `string` | `null` | no | Name of the metric alert (defaults to `alrt-thrpct-<resource-name>`). |
| `resource_group_name` | `string` | n/a | yes | Resource group in which to create the alert. |
| `scopes` | `list(string)` | n/a | yes | List of Search Service resource IDs to monitor. |
| `threshold` | `number` | `10` | no | Percentage threshold. Fires when above this value. |
| `description` | `string` | `null` | no | Custom description. |
| `enabled` | `bool` | `true` | no | Whether the alert is enabled. |
| `action_group_ids` | `list(string)` | `[]` | no | Action Group IDs to notify. |

## 4. Outputs
| Name | Description |
|------|-------------|
| `id` | Resource ID of the metric alert. |
| `name` | Name of the metric alert. |

## 5. Requirements
- Terraform `>= 1.12.1`
- AzureRM provider `>= 4.38.1`
- Existing Resource Group
- Existing Azure AI Search Service
